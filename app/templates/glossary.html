  • Fraud: A transaction that is identified as suspicious or unauthorized.
  • Legit: A legitimate, authorized transaction.
  • Threshold: The minimum fraud probability (%) required to classify a transaction as fraud.
  • SMOTE: A technique used to balance class distribution in imbalanced datasets.
  • Single Prediction: A mode for analyzing a single transaction to instantly detect fraud likelihood.
  • Batch Prediction: A mode where multiple transactions are analyzed at once from a CSV file.
  • Confidence Score: The modelâ€™s certainty (in %) behind its fraud prediction.
  • ROC Curve: A graph showing model performance across different thresholds.
  • Accuracy: The overall percentage of correct predictions made by the model (both fraud and legit).
  • Precision: The ratio of correctly predicted frauds out of all predicted frauds.
  • Recall: The ratio of correctly predicted frauds out of all actual frauds.
  • F1 Score: The harmonic mean of precision and recall; balances false positives and false negatives.
  • AUC Score: Area Under the ROC Curve, representing the model's ability to distinguish between fraud and legit cases.
  • Prediction Log: A saved record containing the prediction result, timestamp, and model used, specific to a user.
  • Input Summary: A condensed view of the transaction data submitted for prediction.
  • Calibrated Model: A model whose output probabilities have been adjusted to better reflect true likelihoods.
  • Random Forest: An ensemble learning method based on multiple decision trees to improve prediction accuracy.
  • XGBoost: A high-performance, tree-based gradient boosting algorithm used for classification.
  • Logistic Regression: A linear model used for binary classification problems like fraud detection.
  • Voting Classifier: An ensemble model combining predictions from multiple models using weighted or majority voting.
